# GitHub

## What Is GitHub?

GitHub is where your code lives online. Think of it as Google Drive for code — it stores your
projects in the cloud, keeps track of every change you make, and lets you share your work with
others. Almost every developer in the world uses it, and it's free for personal projects.

When you're vibe coding, GitHub is where your finished code ends up. Even if an AI writes every
line, you'll want a place to store it, track changes, and deploy it. That place is GitHub.

## Creating an Account

1. Go to [github.com](https://github.com) and click **Sign up**.
2. Enter your email address.
3. Create a password.
4. Choose a username — this becomes part of your profile URL (`github.com/your-username`), so
   pick something you're comfortable sharing.
5. Complete the verification puzzle.
6. Check your email for a verification code and enter it.
7. You'll be asked a few questions about your experience level — answer honestly, it just
   customizes your dashboard.

That's it. You now have a GitHub account.

## Key Concepts

These are the terms you'll hear constantly. You don't need to memorize them — just know they
exist so you can look them up when they come up.

- **Repository (repo):** A project folder on GitHub. It contains all your files, plus the entire
  history of every change ever made to them.
- **Commit:** A saved snapshot of your changes. Every time you commit, Git records exactly what
  changed, when, and a message you write describing why.
- **Branch:** A parallel version of your code. The main branch (called `main`) is the "official"
  version. You create other branches to experiment without affecting `main`.
- **Pull request (PR):** A proposal to merge changes from one branch into another. It's also a
  place for discussion and code review.
- **Issue:** A task, bug report, or feature request attached to a repository. Think of it as a
  to-do item for the project.

## The GitHub Web Interface

When you open a repository on GitHub, here's what you'll see:

- **Code tab** — The default view. Shows all the files and folders in the project. Click any file
  to view its contents.
- **File viewer** — When you click a file, GitHub shows its contents with syntax highlighting.
  You can even edit files directly in the browser (click the pencil icon).
- **Commits history** — Click the clock icon or "X commits" link to see every change ever made
  to the repository. Each commit shows what changed, who changed it, and when.
- **Issues tab** — Where bug reports and feature requests live.
- **Pull requests tab** — Where proposed changes are discussed and merged.
- **Settings tab** — Where the repository owner configures the project.

## Forking vs Cloning

These two actions sound similar but do different things:

- **Cloning** downloads a copy of a repository to your computer. You clone repos so you can work
  on them locally. The connection to the original repo stays intact.
- **Forking** creates your own copy of someone else's repository on GitHub. The copy lives under
  your account. You fork when you want to make changes to a project you don't own.

For most vibe coding projects, you'll create your own repositories and clone them to your
computer. You'll fork when following along with tutorials or contributing to other people's
projects.

## GitHub's Free Tier

GitHub's free plan is generous. You get:

- Unlimited public repositories
- Unlimited private repositories
- Up to 3 collaborators on private repos
- 500 MB of GitHub Packages storage
- GitHub Actions (2,000 minutes/month for CI/CD automation)
- GitHub Pages for hosting static websites

For personal projects and learning, the free tier is more than enough. You won't need to pay
for anything while following this tutorial.

## Useful Settings to Configure

Once your account is set up, consider adjusting these settings (found under your profile icon,
then **Settings**):

- **Profile:** Add a photo and short bio. If you're building a portfolio, this is the first
  thing people see.
- **Notifications:** GitHub sends a lot of emails by default. Under **Notifications**, you can
  switch to "On GitHub" only if the emails get overwhelming.
- **SSH keys:** SSH keys let you push code to GitHub without typing your password every time.
  You can set these up later — GitHub Desktop handles authentication for you in the meantime.
- **Theme:** Under **Appearance**, you can switch between light and dark mode.
